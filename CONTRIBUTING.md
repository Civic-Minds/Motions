# Contributing to Motions

First off, thank you for considering contributing to **Motions**! It's people like you that make it possible to build better civic tools.

## How to Contribute

### Reporting Bugs
If you find a bug, please create an issue on GitHub. Include:
- A clear description of the bug.
- Steps to reproduce the issue.
- Details about your environment (browser, OS, etc.).

### Suggesting Enhancements
We love new ideas! If you have a suggestion for an enhancement:
- Check if it has already been suggested in the issues.
- If not, open a new issue labeled "enhancement".

### Pull Requests
1. Fork the repository.
2. Create a new branch (`git checkout -b feature/your-feature`).
3. Make your changes and commit them with descriptive messages.
4. Push to your branch (`git push origin feature/your-feature`).
5. Open a Pull Request.

## Local Development
To set up the project locally:
```bash
npm install
npm run dev
```

## Community
Join us in our mission to make municipal governance more transparent. If you have questions, feel free to reach out via GitHub Issues.
