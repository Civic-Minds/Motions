# Future Roadmap

This document outlines planned features and architectural improvements for future iterations of the **Motions** platform. These items represent the current vision and are subject to change.

## 📍 Near-Term
- **Interactive Map Integration**: Add a Leaflet.js map to visualize the geographical distribution of motions across city wards.
* **Expanded Data Pipelines**: Support for additional municipal data formats beyond Toronto's TMMIS.
- **Improved Scraper Robustness**: Implement more resilient fetching layers (e.g., Playwright) to handle complex municipal portal architectures.

## 🏗️ Mid-Term
- **User Watchlists**: Allow users to "follow" specific council members or policy topics and receive automated updates.
- **Voting DNA Expansion**: Integrate deeper sentiment analysis of meeting transcripts and speech records.
- **Impact Prediction**: Explore lightweight ML models to project the likely social or financial impact of proposed motions.

## 🚀 Long-Term
- **Scalable Backend API**: Transition from static data stores to a robust relational database and API layer for real-time scaling.
- **Community Commentary**: Integration points for civil discourse and verifiable community input on individual policy items.
- **Open Data Export**: Providing a standardized, public API for researchers and civic organizations to access normalized voting data.
